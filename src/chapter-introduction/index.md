# 第1章：競技プログラミングの世界を知ろう

## 1.1 競技プログラミングって何だろう？

君がScratchでゲームを作ったり、キャラクターを動かしたりした経験を思い出してみよう。あのとき、頭の中で「キャラクターをどう動かそう？」「どんな条件でゴールにたどり着こう？」と考えながら、ブロックを組み合わせていたよね。

競技プログラミングは、まさにその考える楽しさを、より本格的な形で体験できる競技なんだ。

```
【図1-1：ScratchからPythonへの進化】

Scratchのブロック組み合わせ    →    Pythonでのコード記述
┌─────────────────────┐         ┌─────────────────────┐
│ [10回繰り返す]           │         │ for i in range(10):  │
│   [x を 1 ずつ変える]     │    →    │     x += 1          │
│   [x を言う]            │         │     print(x)        │
└─────────────────────┘         └─────────────────────┘
     視覚的・直感的                    論理的・効率的
```

**競技プログラミングの本質**

競技プログラミングとは、制限時間内に出題される問題を、プログラムを書いて解く競技だ。パズルを解く楽しさと、プログラミングの技術が組み合わさった、まったく新しい体験ができる。

```
【図1-2：競技プログラミングの基本構造】

    📝 問題文              👨‍💻 あなた             🖥️ ジャッジシステム
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│「1から100までの   │    │ 💭 考える            │    │ ⚡ コードを実行     │
│ 数で3で割り切れる │ →  │ ↓                  │ →  │ ↓                │
│ 数の合計を求め    │    │ ⌨️ プログラムを      │    │ 📊 結果をチェック   │
│ なさい」          │    │   書く              │    │ ↓                │
└─────────────────┘    └─────────────────┘    │ ✅「正解！」or      │
                                                │ ❌「不正解...」      │
                                                └─────────────────┘
```

**実際の問題解決プロセス**

```
【図1-3：問題解決の思考プロセス（詳細版）】

📋 Step 1: 問題を理解する
┌─────────────────────────────────────────────┐
│ 入力：1から100までの数                         │
│ 条件：3で割り切れる数のみ選ぶ                   │
│ 出力：選んだ数の合計                          │
│                                           │
│ 具体例：3, 6, 9, 12, ..., 96, 99           │
│ ↓                                         │
│ これらを全部足した値を求めればいい！             │
└─────────────────────────────────────────────┘

💡 Step 2: 解法を考える  
┌─────────────────────────────────────────────┐
│ 🔍 方法1: 全部チェック                        │
│ 1から100まで順番に見て、3で割り切れるかチェック   │
│                                           │
│ 🧮 方法2: 数学的に計算                        │
│ 3, 6, 9, ... 99 = 3×(1+2+3+...+33)        │
│                                           │
│ 💭 どちらでも正解！まずは方法1で実装してみよう    │
└─────────────────────────────────────────────┘

⌨️ Step 3: プログラムで実現
┌─────────────────────────────────────────────┐
│ total = 0                                   │
│ for i in range(1, 101):                     │
│     if i % 3 == 0:  # 3で割り切れるか？      │
│         total += i  # 合計に足す             │
│ print(total)        # 答えを出力            │
│                                           │
│ 実行結果: 1683                              │
│ ✅ 正解！                                   │
└─────────────────────────────────────────────┘
```

### 創造性と表現力の開花

```
【図1-9：解法の多様性による創造性の育成】

同じ問題「1から100の和を求める」でも...

解法A: 直接計算アプローチ
┌─────────────────────────────────────────────┐
│ total = 0                                     │
│ for i in range(1, 101):                       │
│     total += i                                │
│ print(total)                                  │
│                                             │
│ 特徴：確実だけど少し遅い                        │
│ 思考：基本に忠実、堅実                         │
└─────────────────────────────────────────────┘

解法B: 数学的アプローチ
┌─────────────────────────────────────────────┐
│ # ガウスの公式を使用                            │
│ n = 100                                       │
│ total = n * (n + 1) // 2                      │
│ print(total)                                  │
│                                             │
│ 特徴：高速で数学的にエレガント                   │
│ 思考：理論活用、効率性重視                      │
└─────────────────────────────────────────────┘

解法C: Pythonらしいアプローチ
┌─────────────────────────────────────────────┐
│ total = sum(range(1, 101))                    │
│ print(total)                                  │
│                                             │
│ 特徴：簡潔で読みやすい                         │
│ 思考：言語の特性活用、可読性重視                │
└─────────────────────────────────────────────┘

解法D: 再帰的アプローチ
┌─────────────────────────────────────────────┐
│ def sum_to_n(n):                              │
│     if n == 1:                                │
│         return 1                              │
│     return n + sum_to_n(n - 1)                │
│ print(sum_to_n(100))                          │
│                                             │
│ 特徴：プログラミング的思考                      │
│ 思考：関数型、再帰的発想                       │
└─────────────────────────────────────────────┘

🎨 君だけの解法を見つけよう！
「同じ答えでも、アプローチは無限大」
```
